# 1. Project Overview

### Project Title  
SFedU Student Event Portal

### Project Idea / Problem Statement  
Southern Federal University (SFedU / ЮФУ) hosts many academic, cultural, and extracurricular events, but information is often scattered across faculty groups, social media, and offline announcements. Students frequently miss events or struggle to register due to the lack of a unified system.

The proposed project aims to create a **centralized web portal** where students and staff can browse upcoming events, register online, and receive digital tickets. The system simplifies event discovery and reduces manual administrative work.

### Target Users or Customers  
- SFedU students attending campus events  
- SFedU staff organizing academic or administrative events  
- Student clubs and organizations hosting activities  
- University departments coordinating participation  

### Business Value  
The portal increases event visibility, improves student engagement, and reduces manual registration tasks. It also provides a structured way to track attendance and distribute event information, supporting more efficient campus communication.

### Project Goal  
To define and prepare an IT solution for a centralized SFedU event portal that enables event browsing, online registration, and digital ticket distribution.

### Main Constraints  
- limited to a web-based solution;  
- no mobile app in the first release;  
- small academic team with limited time and resources;  
- only core features included in version 1;  
- minimal integrations with external systems.

# 3. Project Scope

### In Scope
- web portal for event browsing;  
- event detail pages;  
- online registration;

- attendee list for organizers;  
- email or on-screen confirmation.

### Out of Scope

### Main Deliverables

- stakeholder list;  
- scope statement;  
- high-level requirements;

- milestone schedule.

### Assumptions
- users access the system via web browsers;  
- organizers provide event information manually;  
- digital tickets are simple QR codes or IDs;  
- the first version focuses on core functionality only.

# 4. High-Level Requirements

### Functional Requirements
1. The system shall allow users to browse a list of upcoming SFedU events.  
2. The system shall display detailed information for each event.  
3. The system shall allow students to register for events online.  
4. The system shall generate a digital ticket after successful registration.  
5. The system shall allow organizers to create and manage events.  
6. The system shall allow organizers to view attendee lists.  
7. The system shall send or display registration confirmation.

### Non-Functional Requirements
- the portal must be responsive for desktop and mobile browsers;  
- the interface must be simple and intuitive;  
- the system must handle basic loads typical for campus events;  
- user data must be protected with basic security measures;  
- the system must allow future expansion.

### Main User Roles
- Student  
- Event Organizer  
- Administrator  

### Business Rules
- each event must have a title, date, location, and description;  
- registration is only possible for active events;  
- each student can register once per event;  
- only authorized organizers can create or edit events.

# 5. Use Cases

## Use Case 1: Browse Events
**Actor:** Student  
**Goal:** View upcoming SFedU events  
**Main Flow:**  
1. Student opens the portal.  
2. System displays a list of upcoming events.  
3. Student selects an event to view details.  
4. System shows event information (date, location, description).  
**Outcome:** Student can explore available events.

---

## Use Case 2: Register for an Event
**Actor:** Student  
**Goal:** Register for a selected event  
**Main Flow:**  
1. Student opens an event page.  
2. Student clicks “Register”.  
3. System confirms registration.  
4. System generates a digital ticket.  
**Outcome:** Student receives a valid ticket for the event.

---

## Use Case 3: Create an Event
**Actor:** Event Organizer  
**Goal:** Publish a new event  
**Main Flow:**  
1. Organizer logs into the organizer dashboard.  
2. Organizer selects “Create Event”.  
3. Organizer enters event details.  
4. System saves the event and publishes it.  
**Outcome:** Event becomes visible to students.
