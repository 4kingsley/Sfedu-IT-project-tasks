# Practice 2. Project Initial Documentation  
## SFedU Student Event Portal

---

# 1. Introduction

The purpose of Practice 2 is to prepare the initial project documentation package for the **SFedU Student Event Portal**.  
This documentation expands the project definition created in Practice 1 and formalizes the early project artifacts needed for planning and further development.

The system is intended to centralize event announcements at Southern Federal University (SFedU / ЮФУ), allowing students to browse events, register online, and receive digital tickets. Event organizers will be able to publish events and manage attendee lists.

---

# 2. Project Charter (Short Form)

### **Project Name**  
SFedU Student Event Portal

### **Project Purpose**  
To create a centralized web platform for publishing, browsing, and registering for university events at SFedU.

### **Project Objectives**
- provide a unified event catalog for students and staff;  
- simplify event registration and ticket distribution;  
- reduce manual administrative work for organizers;  
- improve communication and participation tracking.

### **Project Deliverables**
- event browsing interface;  
- event detail pages;  
- online registration module;  
- digital ticket generation;  
- organizer dashboard;  
- attendee list management;  
- initial documentation package.

### **Project Boundaries**
- **In scope:** core event management and registration features.  
- **Out of scope:** mobile app, payments, advanced analytics, external integrations.

### **Project Success Criteria**
- students can browse and register for events;  
- organizers can create and manage events;  
- digital tickets are generated successfully;  
- system is usable on desktop and mobile browsers;  
- documentation supports further practices.

# 4. Scope Statement

### **Project Scope Description**
The system will allow students to browse events, view details, register online, and receive digital tickets. Organizers will be able to create events and manage attendees.

### **In Scope**
- event catalog;  
- event detail pages;  
- registration form;  
- digital ticket generation;  
- organizer dashboard;  
- attendee list export or view;  
- basic notifications.

### **Out of Scope**
- mobile application;  
- payment processing;  
- advanced analytics;  
- integration with SFedU authentication systems;  
- multi-language support.

### **Constraints**
- limited time and academic resources;  
- web-only solution;  
- minimal backend complexity;  
- no external system dependencies.

### **Assumptions**
- organizers manually provide event data;  
- students access the system via browser;  
- digital tickets can be simple codes;  
- version 1 focuses on core functionality.

---

# 5. High-Level Requirements (Extended)

### **Functional Requirements**
1. The system shall allow users to browse upcoming events.  
2. The system shall display event details including date, time, and location.  
3. The system shall allow students to register for events.  
4. The system shall generate a digital ticket upon registration.  
5. The system shall allow organizers to create, edit, and delete events.  
6. The system shall allow organizers to view attendee lists.  
7. The system shall display confirmation messages after registration.  
8. The system shall allow administrators to manage user roles.

### **Non-Functional Requirements**
- responsive design for desktop and mobile browsers;  
- simple and intuitive interface;  
- basic security for user data;  
- stable performance for typical campus traffic;  
- scalable structure for future expansion.

# 8. Initial Project Structure

### **Main Components**
- **Frontend:** event catalogue, event pages, registration UI  
- **Backend:** event management, registration logic, ticket generation

### **Architecture (High-Level)**

- server-side API
