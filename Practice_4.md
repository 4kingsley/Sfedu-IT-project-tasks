# Practice 4. Project Schedule and Cost Estimation  
## SFedU Student Event Portal

---

# 1. Introduction

The purpose of Practice 4 is to create a **project schedule** and perform a **basic cost estimation** for the SFedU Student Event Portal.  
This practice uses the Scrum methodology selected in Practice 3 and transforms the project scope into a structured timeline with estimated effort.

The schedule is simplified for academic purposes and focuses on realistic, achievable milestones for a small student development team.

---

# 2. Project Scheduling Approach

Since the project uses **Scrum**, the schedule is organized into **sprints**.  
Each sprint delivers a functional increment of the system.

### **Assumptions**
- Sprint length: **2 weeks**  
- Total sprints: **4**  
- Team size: **3 students**  
- Workload per student: **20 hours/week**  
- Total team capacity per sprint:  
  - 3 students × 20 hours × 2 weeks = **120 hours per sprint**

# 5. Cost Estimation (Academic Model)

Since this is an academic project, cost estimation is simplified and based on **effort hours**, not real salaries.

### **Assumptions**
- Student developer “hourly rate” (academic model): **$10/hour**  
- Total hours per sprint: **120**  
- Total sprints: **4**  
- Total project hours:  
  - 120 × 4 = **480 hours**

### **Estimated Cost**
- 480 hours × $10/hour = **$4,800 (academic equivalent)**

This is not a real budget — it is a **theoretical cost model** used for project management training.
