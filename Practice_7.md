# Practice 7. Scrum Board and Task Assignment  
## SFedU Student Event Portal

---

# 1. Introduction

The purpose of Practice 7 is to create a **Scrum Board**, assign tasks to team members, and demonstrate how work will be tracked during the development of the SFedU Student Event Portal.

This practice uses the User Stories from Practice 6 and the Sprint structure from Practice 4.  
The Scrum Board is presented in a Markdown-friendly format suitable for GitHub.

---

# 2. Scrum Board Structure

The Scrum Board contains three main columns:

- **To Do** — tasks not yet started  
- **In Progress** — tasks currently being worked on  
- **Done** — completed tasks  

This structure supports transparency and aligns with Scrum principles.

# 8. Workflow Example (Daily Cycle)

1. Team selects tasks from **To Do** during Sprint Planning.  
2. Developers move tasks to **In Progress** when starting work.  
3. Completed tasks move to **Done** after testing.  
4. Team reviews progress during Sprint Review.  
5. Improvements discussed in Sprint Retrospective.
